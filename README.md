# Измерение визуальной метафоры в пространстве CLIP

Проект посвящён анализу, насколько мультимодальная модель CLIP отражает связи между текстовыми выражениями концептуальных метафор и изображениями, связанными с их source- и target-доменами.

В работе используются русскоязычные и англоязычные метафорические выражения, изображения трёх типов (`source`, `target`, `control`), эмбеддинги CLIP, cosine similarity, UMAP-визуализация и векторная арифметика.

## Цель проекта

Исследовать, насколько модель `openai/clip-vit-base-patch32` способна выявлять и отражать семантические связи между компонентами концептуальных метафор в общем текстово-визуальном пространстве представлений.

## Данные

В проекте используются три концептуальные метафоры:

- `time_money` — time is money / время — это деньги
- `argument_war` — argument is war / спор — это война
- `love_journey` — love is a journey / любовь — это путешествие

Для каждой метафоры созданы:

- текстовые фразы на английском и русском языках (по 5 пар для каждой метафоры);
- изображения `source`-домена (10 для каждой метафоры);
- изображения `target`-домена (10 для каждой метафоры);
- контрольные изображения `control`, не связанные напрямую с метафорами (всего 10 изображений для всех метафор).

Изображения были отобраны вручную с Unsplash и Pexels. Текстовые данные основаны на концептуальных метафорах Джорджа Лакоффа и Марка Джонсона.

## Основные гипотезы

**H1.** Метафорические выражения имеют более высокое cosine similarity с изображениями, связанными с соответствующей метафорой (source и target), чем с контрольными изображениями (control).

**H2.** Target-фразы имеют более высокое cosine similarity с изображениями target-домена, чем с изображениями source-домена.

**H3.** Сила метафорического эффекта в пространстве эмбеддингов CLIP различается для русскоязычных и англоязычных текстов.

**H4.** Метафорический эффект будет различаться в зависимости от конкретной метафоры.

**H5.** С помощью векторной арифметики можно сдвинуть абстрактный target-концепт в сторону source-области соответствующей метафоры.

## Используемые методы

В проекте реализованы следующие этапы:

1. Подготовка текстовых и визуальных данных.
2. Загрузка модели `openai/clip-vit-base-patch32`.
3. Получение текстовых эмбеддингов.
4. Получение визуальных эмбеддингов.
5. Расчёт cosine similarity между текстами и изображениями.
6. Сравнение групп изображений: `source`, `target`, `control`.
7. Сравнение результатов для русского и английского языков.
8. Визуализация эмбеддингов с помощью UMAP.
9. Проверка векторной арифметики по схеме:

```text
target_domain - abstract_anchor + source_domain
```

10. Сохранение итоговых таблиц и графиков.

## Основные результаты

По итогам анализа все пять гипотез получили подтверждение:

- метафорические выражения чаще оказываются ближе к связанным с ними изображениям, чем к контрольным;
- target-фразы в среднем лучше соотносятся с target-изображениями, чем с source-изображениями;
- результаты различаются для русского и английского языков;
- сила эффекта зависит от конкретной метафоры;
- векторная арифметика позволяет сдвигать абстрактный target-концепт в сторону source-домена, хотя качество результата зависит от языка, метафоры и набора изображений.

## Ограничения

Основные ограничения проекта:

- анализ включает только три концептуальные метафоры;
- количество изображений ограничено;
- изображения отбирались вручную, поэтому возможна субъективность разметки;
- используется предобученная модель CLIP, которая не была специально обучена для анализа метафор;
- модель CLIP лучше адаптирована к английскому языку, поэтому результаты для русского языка нужно интерпретировать осторожно;
- UMAP используется только как инструмент визуализации и не должен рассматриваться как строгая количественная проверка гипотез.
